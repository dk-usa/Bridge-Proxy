<objective>
Implement tenant-prefixed cache keys and entry validation to prevent cross-tenant data leakage. This establishes defense-in-depth: both key-based isolation (prevention) and entry validation (detection).

Purpose: Security-critical change to ensure cached responses are isolated by tenant. Without this, User from Org A could receive cached response from Org B.

Output: Modified cache service with tenant context, pipeline integration to pass tenant info.
</objective>
